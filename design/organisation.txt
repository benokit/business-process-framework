permissions:
    - api level 
        name: string
        resource: string
        level=1: [ GET 1, PUT 2, POST 3, DELETE 4 ]
    - actor level
        name: string
        hierarchy
        description: string
    - data level
        deny: boolean
        resource: string
        scope: (user, data.ownership) -> maybe data

- api level
    - resource is a string that represents an API url (e.g. entities/party/)
    - higher level implies permissions imply lower layer permissions, e.g. 
        Having permission level POST for a resource implies PUT and GET or the same resource.
        Having permission level GET for a resource is not enough for PUT operations on the resource.
- actor level 
    - permissions are resolved dynamically within an arbitrary context (filters, UI elements, limits)
    - an assigned actor with highest hierarchy number is taken in permission resolution 
        e.g if (user.actor = boss) limit = 1000 else if (user.actor = employee) limit = 100 else limit = 1
            if user.actors = [boss, employee] and boss.hierarchy > employee.hierarchy then user.actor = boss

- row level 
    - permission is associated with a scope, which filters output of a resource
