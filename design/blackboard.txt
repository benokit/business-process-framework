# tooling

folder: implementation/tools
package: @business-framework/tools

purpose: provides tools for discovery, validation, etc 

implement workspace validation:
    For each element resolve corresponding schema for its kind (note that kind has a hierarchical structure with / as delimiter).
    (A schema is a corresponding schema of an element, if the element's kind or any of its parent kinds matches the schema id.)
    Validate elements against corresponding schemas. For failing elements print a useful information with a link to the file 
    and position of the element.
