-------- entity -------

Entity type is a simple concept that defines:

- classification
- identification
- meaning (described in terms of dimensions)
- data model
- states model (states, transitions)

Each entity type permits:

- data versioning
    - data relative diff
    - valid from
- history
    - entity relative diff
    - timestamp 
    - user 

Entity state model is multidimensional and consists of:
    - states: {
        dimA: [stateA_1, stateA_2, ...],
        dimB: [stateB_1, stateB_2, ...],
        ...
    }
    - transactions: vectors (from - to)
        - name
        - from: {
            dimA: stateA,
            dimB: stateB,
            ...
        }
        - to: {
            dim A: stateA',
            dim C: stateC',
            ...
        }

Entity type capabilities are extensible with generic entity components.

Entity components are generic representations or projections of entities, which 
can be used in entity type agnostic contexts.
